// What is Git?

🔹 Distributed — every developer has a full copy of history
🔹 Tracks every change ever made to every file
🔹 Works offline — no internet needed to commit
🔹 Free, open-source, and blazing fast

// The Distinction

Git vs GitHub

Git
• A tool installed on your machine
• Runs on the command line
• Tracks changes locally
• Works offline
• Created 2005 by Linus Torvalds
• Open source, free forever
$ git —version
GitHub
• A platform hosted on the web
• Accessible via browser or CLI
• Stores repos in the cloud
• Requires internet
• Founded 2008, acquired by Microsoft 2018
• Free tier + paid plans
github.com/your-username
Think of it this way: Git is the engine, GitHub is the garage where you park & share

// Building Blocks

4 Core Concepts

Repository
Your project folder tracked by Git. Contains all files + their entire history.
Local or Remote
Commit
A snapshot of your project at a point in time. Has a unique ID (hash) + message.
git commit -m "..."
Branch
A parallel timeline. Work on features without touching the main code. Default: main.
git branch feature
Remote
The version of your repo on GitHub. Push to share, pull to sync with teammates.
origin / upstream
Bonus: Working Directory → Staging Area (Index) → Commit History

// Parallel Development

Branching & Merging

main ──●──●──●──────────────────● ← merge
│                     ↗
feature └──●──●──●──●
               ↑ your work, safely isolated
$ git branch feature-login   # create branch
$ git checkout feature-login  # switch to it
$ git checkout -b new-feature  # create + switch
$ git merge feature-login    # merge back
main is Sacred
Never commit directly to main in team projects
Branches are Cheap
Create one per feature, bug fix, or experiment
Merge Conflicts
Git marks conflicts — you decide which change wins

// Collaborate Like a Pro

GitHub Power Features

Pull Requests
Propose changes from your branch. Teammates review, comment, request changes before merging into main.
Issues
Track bugs, feature requests, and tasks. Assign to team members, add labels, link to PRs. Project management built-in.
Fork & Clone
Fork = your own copy of someone's repo on GitHub. Clone = download a repo to your machine.
GitHub Actions
Automate workflows — test code on every push, deploy on merge. 10.54B CPU minutes used in 2024.
README & Docs
Markdown files shown on your repo homepage. First impression of your project — make it count!
GitHub Pages
Host static websites for free directly from a repo. Perfect for portfolios and project demos!

// Write Pro-Level Code

Best Practices

Commit Often, Push Often
Small, focused commits are easier to review and revert. Don't wait days between commits.
Write Meaningful Messages
BAD: git commit -m "fix"
GOOD: git commit -m "fix: resolve login redirect bug"
Always Use .gitignore
Never commit node_modules, .env files, API keys, or build artifacts. Add them to .gitignore.
Use Feature Branches
One branch per feature or bug. Name clearly: feature/user-auth, fix/nav-crash
Always Write a README
Explain what your project does, how to install it, and how to contribute. It's your repo's homepage.
NEVER do git push --force
On shared branches, force-pushing can destroy teammates' work permanently.
